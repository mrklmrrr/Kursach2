# Демо-сценарий платформы «Мед24» (цифровая клиника)

Цель: за **5–10 минут** показать покупателю полный контур — от лендинга до B2B-метрик.

## 1. Подготовка

1. Запустите MongoDB и backend (`backend/`, порт по умолчанию `5001`).
2. В `backend/.env` задайте `ADMIN_EMAIL`, `ADMIN_PASSWORD` — при первом старте создаётся администратор.
3. Запустите frontend (`frontend/`, `npm run dev`, обычно порт `5173`).
4. Убедитесь, что `VITE_API_URL` указывает на API (или используйте прокси `/api`).

## 2. Роли и вход

| Роль | Где войти | Комментарий |
|------|-----------|---------------|
| **Администратор (владелец клиники)** | `/admin` | Email + пароль из `ADMIN_*` |
| **Врач** | `/login` | Учётная запись врача (часто пароль задаётся при создании в админке, типичный тестовый — `doctor123`) |
| **Пациент** | `/register` или `/login` | Регистрация по телефону |

При **пустой базе** backend создаёт список врачей из сида (`bootstrap`).

## 3. Маршрут для презентации

1. Откройте **`/`** — лендинг с B2B-позиционированием и цифрами выгод.
2. **`/triage`** — выберите симптомы → переход в каталог врачей с фильтром.
3. **`/plans`** — тарифы для клиник и пакеты консультаций (API `/api/platform/plans`).
4. **`/demo`** — краткая памятка по ролям.
5. **`/trust`** — доверие, роли, журнал (описание модели).

**Пациент:** регистрация → главная → запись к врачу → оплата → загрузчик → **видеоконсультация в браузере** (`/consultation/:id`).

**Врач:** кабинет → пациенты → **E-назначение** после приёма.

**Админ:** вкладки **B2B** (конверсия, SLA, e-назначения) и **Соответствие** (журнал аудита, экспорт CSV). Печать отчёта: кнопка «Печать / PDF».

## 4. API для интеграций (кратко)

- `PATCH /api/auth/reminder-preferences` — email/Telegram (пациент).
- `GET /api/prescriptions` — список назначений (пациент).
- `POST /api/doctor/prescriptions` — создать назначение (врач).
- `GET /api/admin/b2b-metrics`, `GET /api/admin/audit-log` — админ.

## 5. Напоминания о приёме (воркер)

После старта backend поднимается **cron каждые 10 минут** (`src/workers/reminderWorker.js`), вызывается `sendAppointmentReminders` (`src/services/reminderDispatch.service.js`).

- Окна: **за ~24 часа** и **за ~1 час** до слота (допуск ±15 мин относительно целевого времени).
- **Email:** нужны `SMTP_*`, `MAIL_FROM` в `.env` и поле **email** у пациента в MongoDB.
- **Telegram:** `TELEGRAM_BOT_TOKEN` и у пациента в профиле сохранён **telegramChatId** (число; удобно получить через @userinfobot или после `/start` вашего бота).
- Повторная отправка не дублируется: коллекция **`ReminderLog`** (уникальная пара `appointmentId` + `kind`).
- Отключить воркер: `REMINDERS_ENABLED=false`.
